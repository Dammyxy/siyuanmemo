# Domain Docs

This repository uses a single-context domain documentation layout.

Before exploring a domain, read `CONTEXT.md` and relevant ADRs under `docs/adr/`. If either is absent, continue silently.

Use terminology defined in `CONTEXT.md` consistently in code, tests, specifications, and issues. Treat missing terminology as a possible domain-modeling task.

Surface conflicts with an existing ADR explicitly instead of silently overriding the decision.
